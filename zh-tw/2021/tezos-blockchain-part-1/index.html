Tezos Blockchain Part I

Created2021-01-26Updated2021-01-26

緣由

在大二上的某天，被教授指派自幹私架的 tezos 區塊鏈，但是老實說我對區塊鏈一點都不熟，就只好硬著頭皮硬幹。
也因為 tezos 是當前區塊鏈較新的技術，所以 document (跟屎一樣) 很分散，需要自己動手挖。

這份紀錄主要應該會分成三個 part。
Part I 是 tezos 到底在幹嘛，該如何架設 carthagenet。
Part II 是 tezos 升級協定的過程還有 tezos 錢包的使用。
Part III 是使用市面上的 blcok explorer 來將區塊鏈上的資料可視化。

小抱怨 🌝

一開始架 tezos 選用的套件只有支援舊的協定，而且其中也有遇到一些 Bug。在 debug 還有實作的時候往往都會發現 tezos 官方 document 跟屎一樣，最後只好直接挖 source code 看還比較快。(雖然 source code 是用 ocaml 實作，真的沒有很理解他的語法就是了)

About Tezos

請自行移駕 Wikipedia XDD。

主要有以下特點

  1. 使用 Proof-of-Stack(PoS) 而不是 Proof-of-Work(PoW)，這可以有效減少 PoW 運算所帶來的能源損耗問題，但是其中也是有一些缺點，其中一個就是有可能造成屯幣的情況
  2. 整個區塊鏈可以透過 Baker (礦工) 的共識來達到協定升級、更新，可以免除 Bitcoin 和 Ethereum 出現硬/軟分岔的問題。
  3. Tezos 自己推出了一個智慧型合約 Michealson，我個人也還在看這部分的資訊，聽說有使用一些驗證，可以降低智慧型合約的漏洞

一些名詞解釋

  • Baker (礦工)
  • Endorsement (關於新區塊的行為)
  • Delegation (將資產委託 Baker 代為挖礦)

Start Node

在這個步驟我們會一次將 node, baker, endorser 一起執行。

執行之前請先注意一下一些事情

  1. 在執行指令時，預設會將 baker 的公私鑰吐出來，請好好存著
  2. --peer 的 ip:port 請依照實際情況調整，只需要指向我們第一個 node 的 RPC port 就好了
  3. 執行完之後吐一堆 error 跟 warning 是正常的，你只需要看到這個訊息 p2p.maintenance: Too few connections (1) ，後面的數字會依照你開起來的 node 增加，如果我們開三個 node，就會有兩個 peer，所以最後可以看到後面會是 (2)

如何使用 tezos

可以從網路上載 tezos-client 也可以直接從 node 裡面複製一份出來用

轉帳

從 a 轉 10000 tz 到 b (b 可以換成 public key)
./tezos-client transfer 10000 from a to b
(如果 b 的帳戶為 0 的話會需要加上 --burn-cap 0.257)

帳戶總額

取得 a 的帳戶總額，也可以換成 public key
./tezos get balance for a
