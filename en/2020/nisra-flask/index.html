Investigate

In the source code, you should notice that it bans function calls and most of the template variables.

BUT NOT ALL OF THEM THO.

According to the Flask Docs, one variable is missing from the blacklist. g.
(I asked the author of this challenge, and turns out this is not the intended solution lol)

Code-Review

Here is the source code of g

In the magic method __repr__ of g, there’s a _app_ctx_stack avaliable in the method.

And, testing on the website, we can get the __repr__ attribute by using g.__repr__.__class__.__dict__

Result:

Notice that we now have __func__ which can get the current function running.

We look in to __func__ by using g.__repr__.__func__.__class__.__dict__

Result:

AND WE GOT __globals__!!

So g.__repr__.__func__.__globals__ should got _app_ctx_stack.

Get Flag

Because we now having _app_ctx_stack, we now can get the app config.
g.__repr__.__func__.__globals__._app_ctx_stack.top.app.config

And because this is not intended solution, we can reuse this payload in flask-deeper.
